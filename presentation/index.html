1. Know your users, know their core goals, understand priorities

If you don't know your users or their core goals, how can you design the right solution for them? Understanding what your main user group is and prioritizing their needs is key.

Image prompt: "A confused detective with a magnifying glass looking for user needs."

2. The path forward should always be clear

A user should never be unsure of how to proceed through your app. It should always be clear what core actions a user has available to them and those should incorporate language and symbology that reinforces what action will be taken.

Image prompt: "A hiker standing at a clear, well-marked trail with signposts."

3. Every action should have a clear reaction

It should always be clear to a user what action has taken place after they interact with the UI. Provide feedback to a user after they've clicked a “save” button to ensure they understand their action has been taken.

Image prompt: "A scientist pressing a button and getting a clear result."

4. Don't lie to your users or make optimistic updates in the UI

Don't show that a value has been changed in the UI if the request to make the change has not actually succeeded yet. Show a loading spinner next to the input to indicate that something is still happening.

Image prompt: "A person nervously watching a loading spinner."

5. Provide clear feedback if there is an issue

It should never be ambiguous or unclear why a user can't progress through your app. Provide helpful message on what fields need to be filled out if a user tries to progress without completing them.

Image prompt: "A user looking at an error message on a computer screen with a magnifying glass."

6. Block as little of the app as possible

While it can be necessary to block parts of the app while a user is waiting for a response, it should be avoided as much as possible. Try to always keep as much of your app usable as you can.

Image prompt: "A user trying to click on blocked buttons on a UI."

7. Identify the main workflow of the app and remove all barriers to achieving that

The app should have a clean, clear, and intuitive UI/UX for achieving the main use case. Any advanced or lesser used options should be hidden away behind menus or alternate UI by default.

Image prompt: "A person swiftly bypassing obstacles on a clearly marked path."

8. Site tours or interactive tutorials are for complex apps and failed UI/UX

If you have to create a tutorial for how to use your “simple” app, you've strayed from the path and need to reevaluate what the core goals/workflows are.

Image prompt: "A frustrated user looking at a complex tutorial."

9. More steps/clicks, more pain

The more steps you add to a user's workflow, the more irksome your app feels and the more likely a user will not realize they have to do something important. Automate or cut down the steps on a cumbersome process wherever possible.

Image prompt: "A user overwhelmed by too many steps on a staircase."

10. UNLESS, you're designing a product for a wide range of users

Then it may make sense to make a more general purpose tool that can fit the use cases for many different kinds of users. Being mindful of your user diversity can lead to inclusive design.

Image prompt: "Diverse group of users finding a common solution."

11. Always get confirmation for destructive or critical actions

If a user triggers an action where the result of that action cannot be undone, ALWAYS ask for confirmation before executing it.

Image prompt: "A user about to press a big red button labeled 'Delete', but there's a pop-up asking 'Are you sure?'."

12. Stand on the shoulders of giants (and steal from them)

The best artists steal from others! It's an inescapable fact that creative expression is, in many ways, just the process of experiencing enough to be able to take pieces of those experiences and combine them to create something new.

Image prompt: "An artist sneakily borrowing elements from different artworks to create a new masterpiece."

13. Don't try and innovate (unless you've got a damned good reason to)

Don't try and reinvent the wheel when it comes to design. A large part of UI/UX design is making something look nice but also feel familiar.

Image prompt: "A user struggling with an 'innovative' wheel design that's not round."
