Ross' (HIGHLY OPINIONATED) UI/UX Tenets

DISCLAIMER

Most of this is in relation to Services projects and less related to Products, but I'll point it out where there could be differences

These are my opinions I've solidified over the years but opinions should always be challenged

What is UI vs UX?

Know your users

If you don't know your users or their core goals, how can you design the right solution for them?

Most of the time, clients or PMs are not your end users.

Understanding what your main user group is and prioritizing their needs is key.

The path forward should always be clear

A user should never be unsure of how to proceed through your app.

It should always be clear what core actions a user has available to them and those should incorporate language and symbology that reinforces what action will be taken.

Not only should you make sure the path is clear, but you should prioritize the path of least resistance for the user.

Every action should have a clear reaction

It should always be clear to a user what action has taken place after they interact with the UI.

Don't just move on.

Provide feedback to a user after they've clicked a “save” button to ensure they understand their action has been taken.

Don't make optimistic updates (for slow processes)

What's an optimistic update?

Don't show that a value has been changed in the UI if the request to make the change has not actually succeeded yet.

Show a loading spinner next to the input to indicate that something is still happening.

I tend to not make optiistic updates anyway since it's an easier pattern to just wait for the success and then refetch the data.

Provide clear feedback if there is an issue

It should never be ambiguous or unclear why a user can't progress through your app.

Provide helpful message on what fields need to be filled out if a user tries to progress without completing them.

Block as little of the app as possible

While it can be necessary to block parts of the app while a user is waiting for a response, it should be avoided as much as possible.

Try to always keep as much of your app usable as you can and utilize localized loading spinners or skeleton components.

Identify the main workflow of the app and remove all barriers to achieving that

The app should have a clean, clear, and intuitive UI/UX for achieving the main use case.

Any advanced or lesser used options should be hidden away behind menus or alternate UI by default.

Site tours or interactive tutorials are for complex workflows and failed UI/UX

If your app is supposed to be simple, the path forward should be clear

If you have to create a tutorial for how to use your “simple” app, you need to reevaluate what the core goals/workflows are.

If your app is simple, and intuitive, it won't need a tour
BUT, if you do need that extra instruction, there's a better way to do it

This is an example I actually love though.

It adds helpful beacons to introduce you to features as you're using the app

Kind of like a much friendlier, more helpful Clippy

More steps/clicks, more pain

The more steps you add to a user's workflow, the more irksome your app feels and the more likely a user will not realize they have to do something important. Automate or cut down the steps on a cumbersome process wherever possible.

UNLESS, you're designing a product for a wide range of users

Then it may make sense to make a more general purpose tool that can fit the use cases for many different kinds of users. Being mindful of your user diversity can lead to inclusive design.

Always confirm destructive or critical actions

If a user triggers an action where the result of that action cannot be undone, ALWAYS ask for confirmation before executing it.

Stand on the shoulders of giants (and steal from them)

The best artists steal from others!

It's an inescapable fact that creative expression is, in many ways, just the process of experiencing enough to be able to take pieces of those experiences and combine them to create something new.

Muzli, Dribble, Figma Community, Code Pen, etc (I'll share links!)

Don't try and innovate (unless you've got a damned good reason to)

Don't try and reinvent the wheel when it comes to design. A large part of UI/UX design is making something look nice but also feel familiar.
